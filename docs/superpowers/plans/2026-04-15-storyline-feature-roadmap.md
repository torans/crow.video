# 故事线功能路线图

> 创建日期：2026-04-15

## 背景

在视频生成领域，"故事线"(Storyline) 是指视频内容的叙事结构——决定视频的开头、中间、结尾各呈现什么内容，如何层层递进引导观众。

当前产品已经实现了完整的视频生成流程，但尚未引入故事线概念。本次讨论旨在评估是否需要引入故事线，以及如何平衡自动化效率与内容可控性。

---

## 当前工作流程分析

### 工作流程概述

```
产品参考 → 文案生成 → 素材匹配 → 视频渲染
```

1. **产品参考**：用户提供产品名称、图片、特征、亮点、目标受众
2. **文案生成**：基于产品信息生成文案（或用户手动输入）
3. **素材匹配**：
   - **智能匹配模式**：基于产品颜色、标签和文案语义自动匹配视频帧
   - **随机模式**：随机选择素材片段
4. **视频渲染**：FFmpeg 合成最终视频

### 优点

- **快速上手** — 用户只需提供产品和文案，其他全自动
- **自动化程度高** — 系统自动分句、自动匹配素材
- **适合量产** — 一套产品文案可以快速生成多个变体

### 缺点

- **可控性低** — 用户无法指定"第5秒要展示产品特写"这类具体需求
- **顺序依赖文案** — 素材顺序完全由文案句子决定，无法独立调整
- **无法复用结构** — 换一套产品/文案，结构完全重新生成，无法复用同一套叙事模板
- **难以精确表达品牌节奏** — 比如"开头3秒吸睛 → 中间10秒产品展示 → 结尾3秒行动号召"这种品牌化节奏无法实现

---

## 故事线工作流程分析

### 工作流程概述

```
故事线模板 → 产品填充 → 素材匹配 → 视频渲染
```

1. **故事线模板**：用户定义视频结构（如：开头吸睛 → 卖点展示 → 行动号召）
2. **产品填充**：将产品信息填入故事线各段落
3. **素材匹配**：按段落语义依次匹配对应素材
4. **视频渲染**：FFmpeg 合成最终视频

### 优点

- **高度可控** — 每个段落都可以指定内容、风格、时长
- **结构可复用** — 同一套故事线模板可以套用到不同产品
- **便于品牌化** — 可以预先定义品牌叙事节奏
- **便于 A/B 测试** — 同一内容换不同故事线结构测试效果

### 缺点

- **学习成本** — 用户需要理解并设计故事结构
- **操作步骤更多** — 需要手动创建/配置段落
- **门槛提高** — 对不懂叙事的用户不友好
- **不适合快速量产** — 每个视频都需要人工规划结构

## 方案对比

| 维度 | 当前工作流程 | 故事线工作流程 |
|------|-------------|---------------|
| 上手难度 | 低 | 中高 |
| 生成速度 | 快（全自动） | 慢（需人工规划） |
| 内容可控性 | 低 | 高 |
| 结构复用性 | 无 | 有 |
| 品牌化支持 | 弱 | 强 |
| 适合用户 | 追求效率的中小商家 | 有品牌化需求的团队 |
| 量产能力 | 强 | 弱 |

---

## 推荐方案：混合模式

保留当前全自动流程作为**"快速生成"模式**，同时提供**"高级模式"**让用户自定义故事线结构。

### 模式选择

- **快速生成模式**：现有全自动流程，适合快速量产
- **故事线模式**：用户自定义叙事结构，适合品牌化内容

### 故事线模板设计（待实现）

用户可预设故事线模板，例如：

```
开场（3s）→ 产品亮点1（8s）→ 产品亮点2（8s）→ 产品亮点3（8s）→ 结尾号召（3s）
```

每个段落支持：
- 自定义名称（如"开场"、"产品展示"）
- 指定时长范围
- 填写该段落对应的文案/关键词
- 指定素材风格（如"特写"、"全景"、"慢动作"）
